# hull increment 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build hull v0.1.0 — a stdlib-only pure engine that renders an injection-safe `.github/workflows/security.yml` (pinned gitleaks secret-scan) from a committed `.hull.json`, plus a `hull:init` skill, dogfooded net-new on shipyard.

**Architecture:** hull is "rigging for security workflows": pure `config → plan → render` engine reusing rigging's `_quote`/`iter_run_blocks`/injection discipline, along a SCANNER-family registry axis. NO `stacks.py`/`detect.py` (secret scanning is language-agnostic). Dogfood = `render(build_plan(load_config(REPO))) == committed security.yml`.

**Tech Stack:** Python 3.9+, stdlib only. pytest.

## Global Constraints

- **Runtime dependencies: stdlib only.** Tests may use pytest.
- **Python 3.9+.** `from __future__ import annotations` where `X | None` appears; no `match`.
- **The engine is pure.** `scanners.py`, `config.py`, `plan.py`, `render.py`, `scaffold.py` MUST NOT import `subprocess`/`os`/networking. An AST `test_purity.py` enforces this.
- **Injection-safe by construction.** The ONLY `${{ ... }}` hull emits is `${{ secrets.GITHUB_TOKEN }}`, and only as an `env:` value on the scanner `uses:` step — never in a `run:`, never `github.*`. `render` has no code path to emit any other expression. Charset validators use `fullmatch` (not `match`+`$`).
- **`.hull.json` keys are camelCase.** `config.load_config(root) -> Optional[Config]` returns `None` when absent, raises `ConfigError` on any invalid-but-present file.
- Skill frontmatter uses **only** `name` and `description`; `name` MUST equal the dir (`init`).
- **No `plugins/hull/tests/__init__.py`** (repo-wide guard forbids `plugins/*/tests/__init__.py`).
- Work on branch `feature/hull` (already created off `main`). Do NOT switch branches. Commit with `git -c user.name="Steve Harmeyer" -c user.email="harmeyersteve@gmail.com"`.
- New plugin → version **0.1.0**.

### Task 4: `render.py` (ported from rigging) + golden + injection test

**Files:** Create `plugins/hull/hull/render.py`, `plugins/hull/tests/golden/security.yml`; Test `plugins/hull/tests/test_render.py`, `plugins/hull/tests/test_injection.py`.

**Interfaces:** Consumes `plan.ScanPlan`/`Job`/`Step`. Produces `render.render(plan) -> str` and `render.iter_run_blocks(yaml_text) -> list[str]` (port rigging's).

**Emitter rules:** read `plugins/rigging/rigging/render.py` and port its `_quote` (double-quote every scalar value) and `iter_run_blocks`. Emit:
```
name: "<name>"
on: [push, pull_request]
permissions:
  contents: read
jobs:
  <job.id>:
    runs-on: "ubuntu-latest"
    steps:
      - uses: "actions/checkout@v4"
        with:
          fetch-depth: "0"
      - uses: "gitleaks/gitleaks-action@v2"
        env:
          GITHUB_TOKEN: "${{ secrets.GITHUB_TOKEN }}"
```
- Every scalar value double-quoted. The ONLY `${{ }}` is the env `${{ secrets.GITHUB_TOKEN }}` (a `with:`/`env:` value, never a run). No run steps in inc 1. Exactly one trailing newline. Deterministic.

- [ ] **Step 1: Hand-author `golden/security.yml`** exactly as the emitter must produce it for the default config. Write `test_render.py`: `render(build_plan(load_config(<default fixture>)))` == the golden byte-for-byte; determinism; every scalar quoted; `permissions: contents: read` and `on: [push, pull_request]` present. Write `test_injection.py` (the 5 assertions from the spec): (1) no registry run step contains `${{`; (2) `iter_run_blocks` finds no `${{` in any run block; (3) every `${{ ... }}` in the output fullmatches `^\$\{\{\s*secrets\.GITHUB_TOKEN\s*\}\}$`; (4) `"github."` absent from output; (5) a hostile `name` is rejected by `load_config` before render.
